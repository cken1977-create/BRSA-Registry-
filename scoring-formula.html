Scoring Formula Specification

The authoritative definition of the FRARI v1.0.0 composite scoring formula. Every readiness score produced by the Legacyline platform is computed using exactly this method.

01

Overview

The FRARI v1.0.0 composite score is a single integer between 0 and 100 representing a participant's overall readiness standing. It is computed deterministically from four weighted sub-scores. The same inputs, evaluated under the same ruleset version, always produce the same output.

This document specifies the exact computation. No deviation from this formula is permitted under FRARI v1.0.0. Any modification to the weighting, normalization method, or input sources requires a new versioned ruleset published by the BRSA Standards Authority.

"The formula is the standard. The standard is the formula. There is no informal version."
02

Inputs

Four inputs are required to compute the composite score. All inputs are derived from structured data in the Legacyline platform. No subjective input is permitted.

Input Source Weight Description
evidence_score evidence_events table · 90-day window 30% Count of evidence events in the last 90 days, capped at 10, normalized to 0–100.
lifecycle_score state_history table · current stage index 25% Current lifecycle stage index (0–5) normalized to 0–100. Certified = 100.
intake_score intake_submissions table · 12 required fields 25% Count of non-null required intake fields present, normalized to 0–100 over 12 fields.
consent_score consents table · status field 20% Granted = 100. Pending = 40. Revoked or absent = 0. No normalization required.
03

Formula

The composite score is computed by applying each weight to its normalized sub-score and summing the results.

composite = (evidence_score × 0.30)
          + (lifecycle_score × 0.25)
          + (intake_score × 0.25)
          + (consent_score × 0.20)
Result rounded to nearest integer · Range: 0–100 · Sealed with SHA-256 at computation time
04

Normalization Rules

All sub-scores must be normalized to a 0–100 scale before weighting. The normalization method for each input is fixed by this specification.

E
Evidence Score Normalization
min(event_count_90d, 10) ÷ 10 × 100. Maximum 10 events. Events older than 90 days are excluded. Each evidence domain counts independently.
L
Lifecycle Score Normalization
stage_index ÷ 5 × 100. Stages: registered=0, data_collecting=1, under_review=2, evaluated=3, certified=4, revoked=0. Revoked returns lifecycle score of 0.
I
Intake Score Normalization
filled_field_count ÷ 12 × 100. Required fields: first_name, last_name, dob, email, phone, address, housing_type, employment_status, monthly_income, monthly_housing_cost, gov_id_uploaded, bank_statement_uploaded.
C
Consent Score — No Normalization Required
Consent is a discrete value. Granted=100, Pending=40, Revoked/Absent=0. Applied directly without normalization.
05

Output & Sealing

The composite score is rounded to the nearest integer and stored in the readiness_snapshots table. At the time of storage, a SHA-256 hash is computed over the canonical input string and stored alongside the score. See BRSA-TECH-001 for the full integrity specification.

The output record contains: composite_score, evidence_score, lifecycle_score, intake_score, consent_score, ruleset_version, score_hash, computed_at. These fields are immutable after writing.

06

Versioning

This specification governs FRARI v1.0.0 only. Every assessment record stores the ruleset version used at computation time. Future versions of the formula will be published as separate specifications with new document IDs. Existing assessments are never recomputed under a new version.

Version history is maintained by the BRSA Standards Authority at brsaregistry.com. All prior versions remain publicly accessible in perpetuity.
